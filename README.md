# コンビニAR

カメラをかざすと周囲がコンビニに変わるモバイルWeb ARアプリ。

**本番URL**: https://hackathonar.vercel.app

## コンセプト

スマホのカメラで身の回りを映すと、AIが「コンビニにありそうな物体」を検出し、検出数に応じて画面がリアルタイムにコンビニ空間へ変身していく。人物を検出した場合は服の色を解析し、セブン-イレブン / ローソン / ファミリーマートのユニフォーム（エプロン）をAR表示する。さらに、値札を店員にドラッグ＆ドロップすることで商品を購入し、PayPayモック決済まで体験できる。

## 技術スタック

| 技術 | 用途 |
|---|---|
| Next.js 16 (App Router) | フレームワーク |
| TypeScript | 型安全 |
| Tailwind CSS | スタイリング |
| TensorFlow.js + COCO-SSD | リアルタイム物体認識（ブラウザ上で実行） |
| Canvas API | 服の色のピクセルサンプリング |
| SVG (インラインReactコンポーネント) | AR演出の描画（エプロン、看板、棚、レジ、QRコード等） |
| Web Audio (HTMLAudioElement) | ブランド別サウンドエフェクト |
| Touch Events API | ドラッグ＆ドロップ購入操作 |
| Vercel | ホスティング・デプロイ |

## アーキテクチャ

```
カメラ映像 (getUserMedia)
    │
    ├─→ COCO-SSD 物体認識 (400ms間隔)
    │       │
    │       ├─→ コンビニ商品判定 → 値札オーバーレイ（ドラッグ可能）
    │       ├─→ 人物検出 → Canvas色解析 → ブランド判定 → ユニフォームSVG
    │       ├─→ コンビニ度スコア算出 → 閾値超えでブランド別サウンド再生
    │       └─→ 店員検出時 → 値札ドラッグのドロップターゲットとして機能
    │
    ├─→ スコアに応じた段階的AR演出 (KonbiniOverlay)
    │
    └─→ 購入フロー
            │
            ├─→ 値札を店員にドラッグ＆ドロップ → カゴに追加
            ├─→ カートパネル（商品一覧・合計金額）
            └─→ PayPay決済モーダル → QRスキャン演出 → レシート表示
```

### コンビニ度スコアの計算

- 検出アイテム数 × 15点（最大60点）
- ユニークカテゴリ数 × 15点（最大40点）
- 合計100点満点

### ユニフォームのブランド判定

1. 人物bboxの胴体中央部分をCanvasに描画
2. ピクセルをサンプリングして平均色をHSLに変換
3. 色相(Hue)でブランドを分類:
   - 赤/オレンジ (H: 0-40, 340-360) → セブン-イレブン
   - 青 (H: 200-260) → ローソン
   - 緑 (H: 80-170) → ファミリーマート
   - 無彩色は明度で判定（明るい→セブン、暗い→ローソン）

### サウンドエフェクト

スコアが閾値（20% / 50% / 80%）を超えると、検出された人物のブランドに応じたサウンドを再生。

| ブランド | サウンドファイル |
|---|---|
| セブン-イレブン | `セブン.mp3` |
| ローソン | `download.mp3` |
| ファミリーマート | `ファミリーマート.mp3` |

- 再生中は新規サウンドを流さない（重複防止）
- スコアが閾値から10%以上下がるとリセットされ、再度トリガー可能

## AR演出の段階

| スコア | 出現する要素 |
|--------|-------------|
| 8%~    | 自動ドアフレーム（アルミサッシ風グラデーション） |
| 12%~   | 蛍光灯（SVG管 + グロー + 光の拡散） |
| 18%~   | 看板（セブン風3色ストライプ + ネオングロー） |
| 25%~   | 左右の商品棚（6段 × カラフルな商品SVG） |
| 30%~   | 床タイル（SVGパターン） |
| 35%~   | レジカウンター（LCD画面、バーコードリーダー、NFC端末、ポイントカード） |
| 45%~   | おでんケース（SVGアニメ湯気付き） |
| 50%~   | 中華まんケース（4種 + 湯気アニメ） |
| 60%~   | コーヒーマシン（HOT/ICEボタン） |
| 62%~   | ATM（画面、カードスロット、テンキー） |
| 80%~   | 全変身エフェクト（金色グロー） |

人物検出時は上記に加えて、ブランド別SVGエプロン + 名札 + 「いらっしゃいませ〜！」吹き出しを表示。

## 購入フロー

1. **人物（店員）を検出** → 値札に緑のヒント「👆 店員にドラッグして購入」が表示
2. **値札をタッチ＆ドラッグ** → ゴーストが指に追従、店員の周囲が緑の破線でハイライト
3. **店員の上でドロップ** → 「✓ カゴに追加！」フィードバックアニメーション
4. **カートパネル** → 画面下部にアイテム一覧・合計金額・クリア/お会計ボタン
5. **「お会計へ →」** → PayPay決済モーダル:
   - **注文確認**: 商品リスト + 合計金額 + 「PayPayで支払う」ボタン
   - **QRスキャン演出**: SVGモックQRコード + 赤いスキャンラインアニメーション
   - **支払い完了レシート**: 店名、日時、商品明細、ポイント付与表示

## 設計方針

- **画像アセット不要**: 全てのAR演出をSVG + CSS + HTMLで描画。外部画像への依存なし
- **モバイルファースト**: `object-fit: cover`でフルスクリーンカメラ、スクロール無効化、`user-scalable=no`、Touch Events対応
- **パフォーマンス考慮**: COCO-SSD `lite_mobilenet_v2` モデル使用、検出は400ms間隔、スコアはスムージング処理
- **段階的演出**: スコアに応じてAR要素を徐々に出現させ、「変身していく」体験を実現
- **座標変換**: 動画の解像度とスクリーンサイズの差を `object-fit: cover` 基準で正しく変換
- **サウンド制御**: 再生中の重複防止、閾値ヒステリシスによるリセット制御
- **インタラクティブ購入**: AR上のドラッグ＆ドロップで物理的な「買い物体験」を実現

## ローカル開発

```bash
npm install
npm run dev
```

http://localhost:3000 をモバイルブラウザで開く（カメラアクセスにはHTTPSが必要なため、実機テストはVercelデプロイ版を推奨）。
